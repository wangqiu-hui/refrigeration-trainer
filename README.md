# 制冷装置操作顺序练习软件：手机版 / PWA 版

这是一个纯前端手机网页项目，用于训练：

- 制冷装置开机操作顺序
- 制冷装置停机操作顺序

特点：

- 不需要数据库
- 不需要后端服务器
- 不需要学生安装 Python
- 手机浏览器可直接使用
- 可发布到 GitHub Pages
- 支持 PWA 添加到手机主屏幕
- 支持离线缓存
- 点击选项区文字后，会把该字母填入当前高亮的作答步骤，而不是自动填入该选项的真实正确步骤

## 本次修正说明（20260430-sequential-fill-fix）

本版本修正了“点击选项文字时直接填入正确步骤”的逻辑错误。

正确逻辑如下：

1. 选项区 A-H 只是被打乱后的步骤候选项。
2. 作答区第 1 步到第 8 步才是学生正在填写的答案位置。
3. 点击选项 B 时，应把 B 填入“当前高亮的作答步骤”。刚进入练习时，当前步骤默认为第 1 步，所以第一次点击 B 会填入第 1 步。
4. 每填完一步，系统会自动把当前步骤切换到下一空白步骤，方便手机端连续点击作答。
5. 如果要修改某一步，先点击作答区中的该步骤，让它变成当前高亮步骤，再点击上方选项文字。
6. 只有点击“确认”时，系统才会用本轮随机生成的正确答案进行判分。

本次代码层面的修正点：

- 删除点击选项时使用真实步骤索引作为填入位置的逻辑。
- 新增 `activeAnswerIndex`，用于记录当前正在作答的步骤。
- 点击选项时只读取被点击按钮的 `data-letter`，并填入当前高亮的答案框。
- 作答区每个步骤可点击/聚焦，以便学生在手机端明确指定要修改的位置。
- 填入后自动切换到下一空白步骤，并用高亮状态提示当前步骤。
- 避免重复选择同一个选项；如需移动已使用的字母，需要先点击目标步骤。
- 更新 `index.html`、`app.js`、`manifest.webmanifest`、`sw.js` 的版本参数和缓存版本，降低手机端旧缓存导致的错觉。

## 本地运行

进入项目目录：

```bash
cd mobile_web
```

启动本地服务：

```bash
python -m http.server 8000
```

电脑浏览器打开：

```text
http://localhost:8000
```

手机测试：

1. 电脑和手机连接同一个 Wi-Fi。
2. 查看电脑局域网 IP，例如 `192.168.1.23`。
3. 手机浏览器打开：

```text
http://192.168.1.23:8000
```

## 发布到 GitHub Pages

推荐把这些文件放到 GitHub 仓库根目录：

```text
index.html
style.css
app.js
steps.json
manifest.webmanifest
sw.js
.nojekyll
icons/
```

然后进入仓库：

```text
Settings → Pages
```

设置：

```text
Source: Deploy from a branch
Branch: main
Folder: /root
```

保存后等待 GitHub Pages 部署完成。

最终网址类似：

```text
https://你的用户名.github.io/仓库名/
```

把这个网址生成二维码，学生扫码即可使用。

## 功能清单

- 首页显示“开机练习”“停机练习”
- 每次进入练习随机打乱 8 个步骤
- 选项显示为 A-H
- 作答区显示第 1 步到第 8 步
- 用户可使用下拉框手动选择 A-H
- 点击选项区文字后，把该字母填入当前高亮的作答步骤
- 初始默认填写第 1 步，填完自动切换到下一空白步骤
- 可点击任意作答步骤后再点击选项文字来修改该步骤
- 动态计算本轮正确答案
- 不把正确答案写死成固定 A-H 字符串
- 未填满时提示“请完成所有步骤后再确认”
- 重复选择时提示“不能重复选择同一个选项”
- 正确时提示“回答正确”
- 错误时显示用户答案、正确答案、错误位置
- 清空按钮可清除所有选择
- 重新开始会重新随机排序
- 返回主界面正常工作
- 手机端自适应布局
- 支持 PWA 离线缓存

## 注意

直接双击 `index.html` 也能看到页面，并且 `app.js` 内置了备用步骤数据；但为了测试 `steps.json`、PWA 和离线缓存，请使用 `python -m http.server 8000` 或部署到 HTTPS 网站。

更新到 GitHub Pages 后，如果个别手机仍显示旧页面，请让学生关闭浏览器标签页后重新扫码；PWA 已添加到桌面的，需要重新打开一次或在浏览器中清除该站点缓存。


## v6 手机端点击可靠性修复

本版针对手机端“点击上方选项文字，下方步骤偶尔不更新”的问题做了移动端优先处理：

- 选项按钮优先使用 `pointerup` 触发填入；不支持 Pointer Events 的旧浏览器回退到 `touchend`。
- `click` 只作为兜底，并通过时间窗口去重，避免移动端合成 click 导致一次点击被处理两次。
- 选项内部文字禁用子元素命中，触摸目标稳定落在按钮本身。
- 选项点击后不再自动 `scrollIntoView` 跳转到作答区，避免手机端滚动动画过程中吞掉下一次触摸。
- `touch-action: manipulation` 用于提示浏览器这是轻点操作，减少移动端手势识别对点击的干扰。
